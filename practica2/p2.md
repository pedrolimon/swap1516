##probar el funcionamiento de la copia de archivos por ssh##
Para probar el funcionamiento por ssh probamos con el comando ssh root@192.168.1.100 (IP fija establecida por mi)
![imagen](https://github.com/pedrolimon/swap1516/blob/master/practica2/ssh.png)
##clonado de una carpeta entre las dos máquinas##
Primero instalo la herramienta 'rsync' con el comando: 'sudo apt-get install rsync'
Después edito el archivo de configuración '/etc/ssh/sshd_config' para cambiar la línea específica a: 'PermitRootLogin yes'
para conectarnos sin problema por ssh como root
Por último basta con ejecutar el comando 'rsync -avz -e ssh root@192.168.1.100':
![imagen](https://github.com/pedrolimon/swap1516/blob/master/practica2/clonado.png)
##configuración de ssh para acceder sin que solicite contraseña##
Para ello debemos configurar un par de claves pública-privada, así ejecutamos el comando 'ssh-keygen -t sda' en la 
máquina secundaria, después copio esta clave en la máquina principal mediante 'ssh-copy-id -i .ssh/id_dsa.pub root@192.168.1.100'
y ya podremos conectarnos con ssh sin necesidad de contraseña:
![imagen]()
##establecer una tarea en cron que se ejecute cada hora para mantener##
Primero edito el archivo '/etc/crontab' y añado la linea '0 * * * * root rsync -avz -e ssh root@192.168.1.100', con esta
línea indicamos que clone la carpeta cada hora:
![imagen](https://github.com/pedrolimon/swap1516/blob/master/practica2/crontab.png)
